# 跨境电商选品详细指南

> 仅适用于跨境模式（Amazon/Shopee/TikTok Shop/Temu）。国内电商请参考 `domestic-guide.md`。

---

## Stage 1：前置收集

收集调研所需的全部输入，避免中途反复追问：

| 信息项 | 必要性 | 说明 |
|:---|:---|:---|
| 产品关键词/ASIN | 必须 | 搜索锚点 |
| 目标市场 | 推荐 | 默认 US，可指定 DE/JP/UK 等 |
| 目标平台 | 推荐 | 默认 Amazon，可选 Shopee/TikTok Shop/Temu/独立站等 |
| 预算范围 | 可选 | 影响入场难度判断 |
| 特定关注点 | 可选 | 如"我特别关心退货率" |

---

## Stage 2：数据采集

按优先级组合采集，不依赖单一数据源：

**采集优先级**：
1. **API 数据**（最快最稳）→ `scripts/maishou_price.py` 获取采购成本参考
2. **CDP 浏览器自动化**（数据最全）→ 需要本地浏览器支持，详见 `cdp-setup.md`
3. **Web Search**（通用兜底）→ 始终可用，但信息可能不够结构化

**采集维度**：
- 竞品价格分布（最低/中位数/最高）
- Review 分析（总量、评分分布、差评关键词提取）
- BSR 排名趋势（大类/小类）
- 卖家数量与类型（FBA/FBM/自营占比）
- 利润链数据（采购成本、头程运费、平台佣金、FBA费用）

数据采集出错时，按降级链处理 → 详见 `fallback-guide.md`

---

## Stage 3：深度分析

对采集数据做多维交叉分析，生成判断：

**分析框架**：
1. **市场吸引力**：市场规模、增长率、季节性
2. **竞争格局**：头部集中度、新卖家进入壁垒、差异化空间
3. **利润可行性**：盈亏平衡点计算
4. **风险因素**：差评共性、合规风险、知识产权、退货率

⚠️ 利润计算是卖家最关心的环节，计算错误会导致严重决策失误——平台佣金和税金基于售价而非成本，这是最常见的坑。

**⛔ 确认门**：分析完成后向用户汇报核心发现，确认方向再写报告。用户说"不用确认直接跑"时可跳过。

**确认门话术示例**：
```
我已完成数据分析，核心发现：
1) 市场规模中等但增速快（月搜索量 XX，同比增长 XX%）
2) 头部集中度低（TOP5 仅占 XX%），新卖家有机会
3) 利润率约 20%，FBA 费可控
4) 主要风险：差评集中在「XXX」，需注意品质控制

方向判断：可以做但需差异化定位。是否继续生成完整报告？如需调整分析角度请告知。
```

---

## Stage 4：报告生成

按 `output-template.md` 完整版格式输出报告，确保：
- 每个判断都有数据支撑，标注来源
- 利润测算展示完整计算过程，不只给结果
- 给出明确建议（做/不做/需进一步验证），不搞模棱两可

---

## 单点查询模式

用户只想算利润或查某一项数据时，直接执行不走完整流程。常见场景：

| 场景 | 操作 |
|:---|:---|
| 利润测算 | 给定采购价和售价，计算 FBA 利润链 |
| BSR 查询 | 给定 ASIN 查排名 |
| 竞品比价 | 给定关键词查 TOP10 价格 |

---

## 监测模式

对已确认关注的品类做持续跟踪：
- 每次执行时对比上次数据，标注变化
- 重点关注：新竞品进入、价格波动、Review增速变化
- 输出增量报告而非全量重复（格式参考 `output-template.md` 监测版）

## 跨境利润计算公式

```
单位硬性成本 = 采购价 + 头程运费 + FBA费 + 采购价×汇率浮动备用金
保本售价 = 单位硬性成本 ÷ (1 - 平台佣金率 - 税率)
利润 = 售价 - 单位硬性成本 - 售价×(平台佣金率 + 税率)
```

⚠️ **关键纠正**：平台佣金和税金都是**基于售价**扣除，不是基于成本。用成本算佣金会导致定价偏低、实际亏钱。

### 示例计算（Amazon US）

```
采购价：¥50 ≈ $7.00
汇率浮动备用金：$7.00 × 3% = $0.21
头程运费：$1.5
FBA费：$5.5（标准尺寸小件）
平台佣金率：15%
税率（VAT/销售税）：0%（US无VAT，部分州有销售税）

单位硬性成本 = $7.00 + $0.21 + $1.5 + $5.5 = $14.21
保本售价 = $14.21 ÷ (1 - 0.15 - 0) = $16.72
若售价 $24.99：
  利润 = $24.99 - $14.21 - $24.99×0.15 = $24.99 - $14.21 - $3.75 = $7.03
  利润率 = $7.03 / $24.99 = 28.1%
```

> ⚠️ **市场对比**：同样商品在 DE 市场（19% VAT），利润差距巨大：

```
同样售价 €24.99（欧盟欧元定价，此处以等值 $24.99 示意）：
税率（VAT）：19%（德国标准税率）
保本售价 = $14.21 ÷ (1 - 0.15 - 0.19) = $14.21 ÷ 0.66 = $21.53
若售价 $24.99：
  利润 = $24.99 - $14.21 - $24.99×(0.15 + 0.19)
       = $24.99 - $14.21 - $8.50 = $2.28
  利润率 = $2.28 / $24.99 = 9.1%

💡 同一件商品，US 市场利润率 28.1%，DE 市场仅 9.1%。VAT 对利润的侵蚀不可忽视！
```

## 采购价层级（从低到高）

| 层级 | 价格水平 | 适用场景 | 可靠度 |
|:---:|:---|:---|:---:|
| 工厂出厂价 | 最低 | 大批量定制（MOQ ≥ 500） | 需验证工厂资质 |
| 1688 批发价 | 中低 | 中小批量拿货（MOQ 10-100） | 较高 |
| 买手平台价 | 中等 | 小批量测款（1-10件起） | 高（实时数据） |
| 淘宝零售价 | 较高 | 仅做参考，不作为采购依据 | 高但无采购意义 |

💡 **建议**：利润测算时用买手价 × 0.8 作为采购成本估算，留出议价和批量折扣空间。
