# 国内电商平台参数提取指南

本文件为国内电商模式（京东/淘宝/拼多多/1688/闲鱼）提供各平台参数提取的详细规则和注意事项。

---

## 京东（jd.com）

### 参数提取规则

⚠️ **2025年起新页面架构变更（截至2026年5月）**：商品详情页采用异步渲染，规格参数不在初始 DOM 中，CDP snapshot 无法读取展开后的内容。

**推荐路线（优先级）**：

| 路线 | 方法 | 适用场景 | 精度 |
|:---|:---|:---:|:---:|
| Route D（推荐） | 访问 ZOL 参数页：`https://detail.zol.com.cn/<ID>/` | 大多数京东商品 | 🟢 高 |
| Route A（备用） | 商品详情 API：`https://d.3.cn/desc/<SKU>` | SKU 已知且有详情页 | 🟡 中 |
| Route C（兜底） | 详情图多模态读取（截图→VLM识别） | 参数在图片中 | 🟠 低 |

**Route D 具体操作**：
```bash
# 从京东商品页 URL 提取 SKU
# URL 格式：https://item.jd.com/<SKU>.html
# 或搜索页 URL 包含 skuId 参数

# 构造 ZOL 查询 URL（通过商品名搜索）
WebFetch "https://detail.zol.com.cn/index.php?c=SearchList&keyword=<商品名>" "提取 ZOL 参数页链接"

# 直接访问 ZOL 参数页
WebFetch "https://detail.zol.com.cn/<product_id>/param.shtml" "提取完整规格参数表"
```

**京东新页面注意事项**：
- 点击"规格参数"tab 后，页面可能跳转异常（agent-browser 元素识别偏差）
- 截图后规格区域为灰色骨架（异步加载占位块），VLM 无法识别
- **结论**：京东商品直接走 Route D，不要在 Route B 上浪费 token

### 价格获取

已登录态下价格可见，通过 CDP snapshot 可获取：
```bash
agent-browser --cdp 9222 snapshot -i | grep -E "¥|价格|plus价|plus会员"
```

未登录态下价格可能显示"登录查看"，需提示用户完成登录。

### 评价获取

```bash
# 评价在商品页底部，需滚动或点击"全部评价"
agent-browser --cdp 9222 snapshot -i | grep -E "好评率|评价数|评价内容"

# 中差评筛选
agent-browser --cdp 9222 click @eN   # 点击"中评"或"差评"tab
```

## 淘宝 / 天猫（taobao.com / tmall.com）

### 参数提取规则

参数信息通常在**详情描述区域**（页面中下部），不是独立的"规格参数"tab。

**推荐路线**：

| 路线 | 方法 | 适用场景 | 精度 |
|:---|:---|:---:|:---:|
| Route B（优先） | CDP snapshot 读取详情描述区域 | 已登录、页面完整加载 | 🟢 高 |
| Route D（补充） | 什么值得买：`https://www.smzdm.com/p/<ID>/` | 热门商品有用户投稿 | 🟡 中 |

**具体操作**：
```bash
# 滚动到详情描述区域
agent-browser --cdp 9222 eval "window.scrollTo(0, document.body.scrollHeight/2)"

# 读取详情区域
agent-browser --cdp 9222 snapshot -i | grep -E "参数|规格|品牌|型号|尺寸|重量|材质"
```

### 登录要求

淘宝/天猫**必须登录**才能查看完整参数和价格，未登录会：
- 价格显示"登录后查看"或模糊处理
- 参数区域内容被截断
- 评价只显示部分

> 💡 **提示用户**：在启动的浏览器中登录淘宝/天猫账号，完成后再通知我继续。

## 拼多多（yangkeduo.com）

### 参数提取规则

拼多多商品页为**移动端页面**（即使 PC 浏览器访问也会跳转到移动端 URL），参数信息偏少。

**推荐路线**：

| 路线 | 方法 | 适用场景 | 精度 |
|:---|:---|:---:|:---:|
| Route B（优先） | CDP snapshot 读取移动端页面 | 已登录 | 🟡 中 |
| Route D（补充） | 什么值得买 / 慢慢买 | 热门商品 | 🟡 中 |

**具体操作**：
```bash
# 拼多多商品页 URL 格式
# https://mobile.yangkeduo.com/goods2.html?goods_id=<ID>

# 页面为移动端渲染，snapshot 可能不完整
# 建议切换移动端 UA 后再试
agent-browser --cdp 9222 eval "navigator.userAgent"
```

### 注意事项

- 拼多多页面大量使用图片展示参数（非文字 DOM），提取难度大
- 评价内容相对简单，中差评分析价值有限
- **更适合**：价格对比、销量参考，不适合深度参数对比

## 1688（1688.com）

### 参数提取规则

1688 为批发平台，参数信息较全，但页面结构与零售平台不同。

```bash
# 1688 商品页 URL 格式
# https://detail.1688.com/offer/<ID>.html

# 参数在"商品详情"区块
agent-browser --cdp 9222 snapshot -i | grep -E "参数|规格|起批量|批发价"
```

### 特色字段

| 字段 | 说明 |
|:---|:---|
| 起批量 | 最低批发数量 |
| 批发价阶梯 | 不同数量的单价 |
| 加工定制 | 是否支持 OEM/ODM |
| 产地 | 工厂位置 |

## 闲鱼（xianyu.com）

### 参数提取规则

⚠️ **买手 API 不支持闲鱼**，需通过 CDP 直接访问。

闲鱼为**二手交易平台**，商品信息由卖家手动填写，格式不统一。

```bash
# 闲鱼商品页 URL 格式
# https://www.xianyu.com/item.htm?id=<ID>

# 参数在商品描述区域（非结构化）
agent-browser --cdp 9222 snapshot -i | grep -E "品牌|型号|成色|购买时间|配件"
```

### 注意事项

- 参数信息不完整（依赖卖家自觉填写）
- 价格由卖家自定义，无统一标准
- **更适合**：查看二手市场行情，不适合做严谨的参数对比

## 国内利润计算公式

### 基础公式

```
利润 = 售价 - (采购价 + 运费 + 平台扣点)
利润率 = 利润 ÷ 售价 × 100%
```

### 各平台扣点参考（截至2026年5月）

| 平台 | 佣金率 | 说明 |
|:---|---:|:---|
| 京东自营 | ~10-15% | 扣除点 + 仓储费 |
| 京东 POP | ~3-8% | 各类目不同 |
| 淘宝 | ~0-5% | 基础无佣金，营销工具另计 |
| 天猫 | ~2-5% | 各类目不同 |
| 拼多多 | ~0.6% | 支付手续费，无佣金 |
| 1688 | ~2-5% | 批发平台佣金较低 |

> ⚠️ **注意**：平台扣点可能调整，以上为参考值。计算利润时建议查询最新官方费率。

> 💡 **税率说明**：国内平台售价通常已含增值税（如 13%），利润测算时不需要额外扣除。但天猫国际、考拉海购等**跨境进口平台**可能收取跨境电商综合税（关税+增值税+消费税），计算时应额外加入税率项。跨境出口场景请使用 `profit_calc.py crossborder` 模式。

### 示例计算

```
商品：某电脑机箱
采购价（1688）：¥80
运费：¥8
平台扣点（京东 POP 5%）：售价 × 0.05
售价：¥159

利润 = 159 - (80 + 8 + 159×0.05) = 159 - (88 + 7.95) = ¥63.05
利润率 = 63.05 ÷ 159 × 100% = 39.7%
```

## 数据来源标注规范

输出时每个字段必须标注数据来源：

| 标注 | 含义 | 使用场景 |
|:---:|:---|:---|
| 🟢 高 | API/已登录页面直读 | 买手 API、CDP 登录态 |
| 🟡 中 | 第三方站补充 | ZOL、什么值得买 |
| 🟠 低 | 图片多模态读取 | 参数为图片形式 |
| 🔴 手动 | 用户手动提供 | API 和 CDP 均失败 |
| ❓ 未检索到 | 所有方法均未获取到 | 明确标注，不猜测 |
