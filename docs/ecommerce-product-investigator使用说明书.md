> **技能版本**：v0.5.0（2026-05-29更新）

## 📋 一、技能概述

这是一个 **AI 驱动的电商商品调研与选品分析工具**，支持双模式：

| 模式 | 目标用户 | 核心功能 |
|------|---------|---------|
| **国内电商模式** | 消费者 | 全网比价、参数对比、评价分析、AI购买建议 |
| **跨境电商模式** | 卖家 | 选品分析、竞品调研、利润测算、市场风险评估 |

**当前版本**：v0.5.0（2026-05-29更新）  
**支持平台**：京东、淘宝、拼多多、闲鱼、1688、Amazon、Shopee、TikTok Shop、Temu

## 🔄 三、工作流程详解

我支持 **4 种工作模式**，会根据你的需求自动选择：

---

### 🚀 模式1：快速模式（~3轮对话）

**适用场景**：快速查看产品基本面

#### 你的操作：
- 提供产品关键词或链接
- 可选：指定目标市场（默认US）

#### 我的工作流程：

```
Step 1: 调用 maishou_price.py 获取采购价参考
Step 2: Web Search 搜索竞品（价格区间、评分、Review数量）
Step 3: 输出快速摘要（市场容量、竞争程度、入场难度）
```

#### 示例：

```
你：「帮我看看 Amazon US 上 dog leash 的竞争情况」
我：识别为快速模式 → 采集数据 → 输出摘要
```

**输出示例**：
- 价格区间：$8-25
- Review 中位数：2000+
- 竞争程度：高

> ⚠️ 如果你想深入，我会自动升级为完整调研模式。

---

### 🔍 模式2：完整调研（~8轮对话）

**适用场景**：深度分析，做投资决策

#### Stage 1 - 前置收集（我会问你）

| 信息项 | 是否必须 | 说明 |
|--------|---------|------|
| 产品关键词/ASIN | ✅ 必须 | 搜索锚点 |
| 目标市场 | � 권장 推荐 | 默认 US，可指定 DE/JP/UK |
| 目标平台 | � 권장 推荐 | 默认 Amazon |
| 预算范围 | ❌ 可选 | 影响入场难度判断 |
| 特定关注点 | ❌ 可选 | 如"特别关心退货率" |

#### Stage 2 - 数据采集（我自动执行）

**采集优先级**：
1. **API 数据**（最快最稳）→ `scripts/maishou_price.py` 获取价格参考
2. **CDP 浏览器自动化**（数据最全）→ 需要本地浏览器支持
3. **Web Search**（通用兜底）→ 始终可用，但信息可能不够结构化

**采集维度**：
- 竞品价格分布（最低/中位数/最高）
- Review 分析（总量、评分分布、差评关键词提取）
- BSR 排名趋势（大类/小类）
- 卖家数量与类型（FBA/FBM/自营占比）
- 利润链数据（采购成本、头程运费、平台佣金、FBA费用）

> 💡 数据采集出错时，按降级链处理 → 详见 `references/fallback-guide.md`

#### Stage 3 - 深度分析（我自动执行，然后向你汇报）

**分析框架**：

1. **市场吸引力**：市场规模、增长率、季节性
3. **竞争格局**：头部集中度、新卖家进入壁垒、差异化空间
4. **利润可行性**：盈亏平衡点计算
5. **风险因素**：差评共性、合规风险、知识产权、退货率

⚠️ **确认门**：分析完成后我会向你汇报核心发现，确认方向再写报告。

#### Stage 4 - 报告生成（我自动执行）

按 `references/output-template.md` 完整版格式输出报告，确保：
- 每个判断都有数据支撑，标注来源
- 利润测算展示完整计算过程，不只给结果
- 给出明确建议（做/不做/需进一步验证），不搞模棱两可

---

- **BSR 查询**：给定 ASIN 查排名

- 输出增量报告而非全量重复（格式参考 `references/output-template.md` 监测版）

## 💡 四、核心原则

你需要了解这些原则，才能理解我的输出：

| 原则 | 说明 |
|------|------|
| **数据可追溯** | 每个事实性陈述标注来源，无来源数据不如没有 |
| **CDP 优先但不强求** | 连接失败时自动降级到搜索模式，不阻塞流程 |
| **利润计算用售价做基数** | ⚠️ 平台佣金和税金从售价扣除，不是从成本扣除（新手最常犯的错误） |
| **平台差异要尊重** | Amazon 和 Shopee 的佣金结构、Review机制、BSR算法完全不同 |
| **降级优于卡死** | 数据源不可用时自动切换，80%及时数据比100%永远等不到的数据有用 |

## 💰 五、利润计算公式（重要！）

### 国内版：

```
利润 = 售价 - (采购价 + 运费 + 平台扣点)
```

### 跨境版（⚠️ 注意）：

```
单位硬性成本 = 采购价 + 头程运费 + FBA费 + 采购价×汇率浮动备用金
保本售价 = 单位硬性成本 ÷ (1 - 平台佣金率 - 税率)
利润 = 售价 - 单位硬性成本 - 售价×(平台佣金率 + 税率)
```

**常见错误**：用成本算佣金 → 定价偏低 → 实际亏钱

#### 计算示例：

```
采购价：¥50 ≈ $7.00
汇率浮动备用金：$7.00 × 3% = $0.21
头程运费：$1.5
FBA费：$5.5（标准尺寸小件）

单位硬性成本 = $7.00 + $0.21 + $1.5 + $5.5 = $14.21
保本售价 = $14.21 ÷ (1 - 0.15 - 0) = $16.72
若售价 $24.99：
  利润 = $24.99 - $14.21 - $24.99×0.15 = $24.99 - $14.21 - $3.75 = $7.03
  利润率 = $7.03 / $24.99 = 28.1%
```

## ⚖️ 六、合规提醒

### ✅ 允许的行为：
- 采集公开可见的商品名称、价格、参数
- 通过官方API获取授权范围内的数据
- 分析自己店铺的运营数据
- 控制采集频率，遵守平台robots.txt规则

### ❌ 禁止的行为：
- 绕过验证码/登录墙抓取非公开数据
- 高频批量爬取导致平台服务器压力过大
- 抓取其他商家的后台运营数据
- 将采集的数据转售或用于未授权营销
